     602                 :            :  /** @brief n-dimensional dense array class \anchor CVMat_Details
     603                 :            : 
     604                 :            : The class Mat represents an n-dimensional dense numerical single-channel or multi-channel array. It
     605                 :            : can be used to store real or complex-valued vectors and matrices, grayscale or color images, voxel
     606                 :            : volumes, vector fields, point clouds, tensors, histograms (though, very high-dimensional histograms
     607                 :            : may be better stored in a SparseMat ). The data layout of the array `M` is defined by the array
     608                 :            : `M.step[]`, so that the address of element \f$(i_0,...,i_{M.dims-1})\f$, where \f$0\leq i_k<M.size[k]\f$, is
     609                 :            : computed as:
     610                 :            : \f[addr(M_{i_0,...,i_{M.dims-1}}) = M.data + M.step[0]*i_0 + M.step[1]*i_1 + ... + M.step[M.dims-1]*i_{M.dims-1}\f]
     611                 :            : In case of a 2-dimensional array, the above formula is reduced to:
     612                 :            : \f[addr(M_{i,j}) = M.data + M.step[0]*i + M.step[1]*j\f]
     613                 :            : Note that `M.step[i] >= M.step[i+1]` (in fact, `M.step[i] >= M.step[i+1]*M.size[i+1]` ). This means
     614                 :            : that 2-dimensional matrices are stored row-by-row, 3-dimensional matrices are stored plane-by-plane,
     615                 :            : and so on. M.step[M.dims-1] is minimal and always equal to the element size M.elemSize() .
     616                 :            : 
     617                 :            : So, the data layout in Mat is compatible with the majority of dense array types from the standard
     618                 :            : toolkits and SDKs, such as Numpy (ndarray), Win32 (independent device bitmaps), and others,
     619                 :            : that is, with any array that uses *steps* (or *strides*) to compute the position of a pixel.
     620                 :            : Due to this compatibility, it is possible to make a Mat header for user-allocated data and process
     621                 :            : it in-place using OpenCV functions.
     622                 :            : 
     623                 :            : There are many different ways to create a Mat object. The most popular options are listed below:
     624                 :            : 
     625                 :            : - Use the create(nrows, ncols, type) method or the similar Mat(nrows, ncols, type[, fillValue])
     626                 :            : constructor. A new array of the specified size and type is allocated. type has the same meaning as
     627                 :            : in the cvCreateMat method. For example, CV_8UC1 means a 8-bit single-channel array, CV_32FC2
     628                 :            : means a 2-channel (complex) floating-point array, and so on.
     629                 :            : @code
     630                 :            :     // make a 7x7 complex matrix filled with 1+3j.
     631                 :            :     Mat M(7,7,CV_32FC2,Scalar(1,3));
     632                 :            :     // and now turn M to a 100x60 15-channel 8-bit matrix.
     633                 :            :     // The old content will be deallocated
     634                 :            :     M.create(100,60,CV_8UC(15));
     635                 :            : @endcode
     636                 :            : As noted in the introduction to this chapter, create() allocates only a new array when the shape
     637                 :            : or type of the current array are different from the specified ones.
     638                 :            : 
     639                 :            : - Create a multi-dimensional array:
     640                 :            : @code
     641                 :            :     // create a 100x100x100 8-bit array
     642                 :            :     int sz[] = {100, 100, 100};
     643                 :            :     Mat bigCube(3, sz, CV_8U, Scalar::all(0));
     644                 :            : @endcode
     645                 :            : It passes the number of dimensions =1 to the Mat constructor but the created array will be
     646                 :            : 2-dimensional with the number of columns set to 1. So, Mat::dims is always \>= 2 (can also be 0
     647                 :            : when the array is empty).
     648                 :            : 
     649                 :            : - Use a copy constructor or assignment operator where there can be an array or expression on the
     650                 :            : right side (see below). As noted in the introduction, the array assignment is an O(1) operation
     651                 :            : because it only copies the header and increases the reference counter. The Mat::clone() method can
     652                 :            : be used to get a full (deep) copy of the array when you need it.
     653                 :            : 
     654                 :            : - Construct a header for a part of another array. It can be a single row, single column, several
     655                 :            : rows, several columns, rectangular region in the array (called a *minor* in algebra) or a
     656                 :            : diagonal. Such operations are also O(1) because the new header references the same data. You can
     657                 :            : actually modify a part of the array using this feature, for example:
     658                 :            : @code
     659                 :            :     // add the 5-th row, multiplied by 3 to the 3rd row
     660                 :            :     M.row(3) = M.row(3) + M.row(5)*3;
     661                 :            :     // now copy the 7-th column to the 1-st column
     662                 :            :     // M.col(1) = M.col(7); // this will not work
     663                 :            :     Mat M1 = M.col(1);
     664                 :            :     M.col(7).copyTo(M1);
     665                 :            :     // create a new 320x240 image
     666                 :            :     Mat img(Size(320,240),CV_8UC3);
     667                 :            :     // select a ROI
     668                 :            :     Mat roi(img, Rect(10,10,100,100));
     669                 :            :     // fill the ROI with (0,255,0) (which is green in RGB space);
     670                 :            :     // the original 320x240 image will be modified
     671                 :            :     roi = Scalar(0,255,0);
     672                 :            : @endcode
     673                 :            : Due to the additional datastart and dataend members, it is possible to compute a relative
     674                 :            : sub-array position in the main *container* array using locateROI():
     675                 :            : @code
     676                 :            :     Mat A = Mat::eye(10, 10, CV_32S);
     677                 :            :     // extracts A columns, 1 (inclusive) to 3 (exclusive).
     678                 :            :     Mat B = A(Range::all(), Range(1, 3));

     680                 :            :     // that is, C \~ A(Range(5, 9), Range(1, 3))

     684                 :            :     // size will be (width=10,height=10) and the ofs will be (x=1, y=5)
     685                 :            : @endcode
     686                 :            : As in case of whole matrices, if you need a deep copy, use the `clone()` method of the extracted
     687                 :            : sub-matrices.
     688                 :            : 
     689                 :            : - Make a header for user-allocated data. It can be useful to do the following:
     690                 :            :     -# Process "foreign" data using OpenCV (for example, when you implement a DirectShow\* filter or

     699                 :            :     @endcode
     700                 :            :     -# Quickly initialize small matrices and/or get a super-fast element access.
     701                 :            :     @code

     703                 :            :         Mat M = Mat(3, 3, CV_64F, m).inv();
     704                 :            :     @endcode
     705                 :            :     .
     706                 :            : 
     707                 :            : - Use MATLAB-style array initializers, zeros(), ones(), eye(), for example:
     708                 :            : @code
     709                 :            :     // create a double-precision identity matrix and add it to M.
     710                 :            :     M += Mat::eye(M.rows, M.cols, CV_64F);
     711                 :            : @endcode
     712                 :            : 
     713                 :            : - Use a comma-separated initializer:
     714                 :            : @code
     715                 :            :     // create a 3x3 double-precision identity matrix
     716                 :            :     Mat M = (Mat_<double>(3,3) << 1, 0, 0, 0, 1, 0, 0, 0, 1);
     717                 :            : @endcode
     718                 :            : With this approach, you first call a constructor of the Mat class with the proper parameters, and
     719                 :            : then you just put `<< operator` followed by comma-separated values that can be constants,
     720                 :            : variables, expressions, and so on. Also, note the extra parentheses required to avoid compilation
     721                 :            : errors.

     724                 :            : array header is built on top of user-allocated data, you should handle the data by yourself. The
     725                 :            : array data is deallocated when no one points to it. If you want to release the data pointed by a
     726                 :            : array header before the array destructor is called, use Mat::release().

     730                 :            : formula directly in the code. If you know the array element type (which can be retrieved using the
     731                 :            : method Mat::type() ), you can access the element \f$M_{ij}\f$ of a 2-dimensional array as:
     732                 :            : @code
     733                 :            :     M.at<double>(i,j) += 1.f;

     739                 :            : the row first, and then just use the plain C operator [] :
     740                 :            : @code
     741                 :            :     // compute sum of positive matrix elements
     742                 :            :     // (assuming that M is a double-precision matrix)
     743                 :            :     double sum=0;
     744                 :            :     for(int i = 0; i < M.rows; i++)
     745                 :            :     {
     746                 :            :         const double* Mi = M.ptr<double>(i);
     747                 :            :         for(int j = 0; j < M.cols; j++)
     748                 :            :             sum += std::max(Mi[j], 0.);
     749                 :            :     }
     750                 :            : @endcode
     751                 :            : Some operations, like the one above, do not actually depend on the array shape. They just process
     752                 :            : elements of an array one by one (or elements from multiple arrays that have the same coordinates,
     753                 :            : for example, array addition). Such operations are called *element-wise*. It makes sense to check
     754                 :            : whether all the input/output arrays are continuous, namely, have no gaps at the end of each row. If
     755                 :            : yes, process them as a long single row:
     756                 :            : @code
     757                 :            :     // compute the sum of positive matrix elements, optimized variant
     758                 :            :     double sum=0;
     759                 :            :     int cols = M.cols, rows = M.rows;
     760                 :            :     if(M.isContinuous())
     761                 :            :     {
     762                 :            :         cols *= rows;
     763                 :            :         rows = 1;
     764                 :            :     }
     765                 :            :     for(int i = 0; i < rows; i++)
     766                 :            :     {
     767                 :            :         const double* Mi = M.ptr<double>(i);
     768                 :            :         for(int j = 0; j < cols; j++)
     769                 :            :             sum += std::max(Mi[j], 0.);
     770                 :            :     }
     771                 :            : @endcode
